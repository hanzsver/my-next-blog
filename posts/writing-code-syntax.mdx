코드를 작성하는 표기법에는 `카멜, 파스칼, 헝가리안, 스네이크, 케밥` 등의 다양한 케이스가 있습니다. 프로그램의 언어마다, 개인의 편한 방식마다 모두 다르게 사용되지만 협업을 하는 경우에는 서로 규칙을 정하여 프로젝트를 수행해야 일의 효율을 훨씬 높일 수 있을 것입니다.

이 글은 잘 사용되지 않거나 지양되는 표기법을 제외한
`카멜, 파스칼, 스네이크,케밥` 네 가지 표기법에 대해 정리합니다.

---

## 1. 카멜 표기법 (Camel Case)

처음은 **소문자**로 시작하며, 두 번째 단어부터는 **대문자**로 시작하는 표기법입니다.
첫번째로 시작하는 단어는 **동사**로 시작하는 것을 권장합니다.

```java
currentPage: 0
totalCount: 11
```

## 2. 파스칼 표기법 (Pascal Case)

카멜 표기법과 비슷하나 다른 점이 있습니다.
파스칼 표기법은 첫 단어부터 **대문자**로 표기합니다.

```java
CurrentPage: 0
TotalCount: 11
```

## 3. 스네이크 표기법 (Snake Case)

단어 사이 사이 **언더바(`_`)**를 넣어 표기합니다.

```java
current_page: 0
total_count: 11
```

## 4. 케밥 표기법 (Kebab Case)

스네이크 표기법과 비슷하나 다른 점이 있습니다.
케밥 표기법은 언더바가 아닌 **하이픈(`-`)**으로 표기합니다.

```java
current-page: 0
total-count: 11
```

## 5. 헝가리안 표기법 (Kebab Case) (첨언자: 최정환 책임님)

스네이크 표기법과 비슷하나 다른 점이 있습니다.
케밥 표기법은 언더바가 아닌 **하이픈(`-`)**으로 표기합니다.

```java
current-page: 0
total-count: 11
```
